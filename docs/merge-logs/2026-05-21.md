### Observations

- **Install drift surfaced at exactly the wrong time.** Both `post_merge_cleanup.py` (from merge-pull-requests) and `--post-merge` mode (in cleanup-feature) were absent from the installed copies in this repo because the last skill sync predated them in the canonical source. The skill's `--post-merge` handoff contract is fine; the failure mode is that the orchestrator (`/merge-pull-requests`) renders an approval prompt for a flag the next-step skill won't understand until you re-sync. Suggests a "skill version check at handoff time" would be cheap insurance — or simpler, the post_merge_cleanup probe could detect a missing `--post-merge` capability in the target skill and surface that to the operator before the approval prompt.
- **Network glitch hit `gh` GraphQL but not REST.** Mid-run, `gh pr view` (which uses GraphQL) timed out for ~30s; same time window, `curl https://api.github.com/zen` returned in 79ms and `gh api repos/.../pulls/40` worked fine. The connection-pool / DNS state in the gh client is more fragile than the underlying network. `merge_pr.py`'s first attempt failed in the middle of `_try_merge_queue` (GraphQL post-status check), leaving the PR in `state=open, merged=false` — clean rollback, no inconsistent state. Retry succeeded.
- **PR body as Change Summary substitute.** The skill template expects `CHANGES MADE / DIDN'T TOUCH / CONCERNS` in the PR body. PR #40's autopilot-generated body went well beyond that template — `## Summary`, `## Evidence Trail` (with per-phase finding counts and disposition), `## Architecture impact`, `## Notable bugs caught by multi-vendor review` (the CONCERNS equivalent), `## Out-of-scope` (the DIDN'T TOUCH equivalent), `## Convergence Report`. The bodies autopilot generates already satisfy the spirit of the template, just with different section headers. Worth noting that the skill's gate ("PR is missing CHANGES MADE section → flag description-incomplete") would false-positive on this style.
