### Observations

- **The GitHub API file cap produced a wrong intermediate conclusion.** `gh pr diff 43 --name-only` silently truncates at roughly 300 files. At 402 files, `pyproject.toml` fell outside the returned set, so an earlier check concluded that #43 touched no `pyproject.toml` and therefore could not contain the gen-eval fix. That was reported to the operator as fact and was wrong: #43 did remove the gen-eval local path dependency. Verified after the merge with `git diff --name-only`, which has no cap and reported 402 files. Lesson: for any PR near or above 300 files, use git locally rather than the PR diff API. The 2026-05-21 log had already flagged this cap for PR #40, so the precedent existed and was not applied.
- **Rebase is unavailable for any branch containing merge commits.** GitHub refuses the whole operation up front rather than partially applying it, because rebase requires replaying each commit as a linear sequence and merge commits have two parents. #43 had 11. Worth checking `gh pr view <n> --json commits` for merge commits before offering rebase as a strategy, so the operator is not asked to choose an impossible option.
- **Main CI is green again for the first time since 2026-06-10.** Two independent breakages had to clear together: the gen-eval local path dependency (`{ path = "../agentic-coding-tools/packages/gen-eval" }`), which #43 removed, and the relative `ASSISTANT_PERSONAS_DIR`, fixed in this session. Both had the same underlying shape, a path that resolves only in one specific environment.
- **The relative path bug reproduced deterministically once the mechanism was understood.** Job-level env in `ci.yml` makes conftest's `os.environ.setdefault` a no-op, so the workflow value is authoritative; the CLI tests then chdir via `CliRunner.isolated_filesystem()` and the relative path resolves against a temp directory. Relative gives 10 failed / 12 passed, absolute gives 22 passed. Anchoring to `${{ github.workspace }}` fixed it without touching test logic or src.
- **The cleanup-feature skill assumes repo wiring that does not exist here.** Step 4 calls `make architecture` and Step 6 calls `make decisions`, but this repository has no `Makefile` at all. There is also no `validate-decision-index` CI job, so the failure mode the skill warns about is absent. Inspection showed `docs/decisions/` holds hand-authored retroactive ADRs rather than a derived index, so running the generator would have risked overwriting authored prose. The step was skipped deliberately and recorded in the session log rather than forced.
- **Four unrelated `.claude/worktrees/agent-*` worktrees still remain**, unchanged since the 2026-05-21 session flagged them. Out of scope for change-scoped cleanup, which only removes remnants for the change-id being archived. Still worth a deliberate sweep.
